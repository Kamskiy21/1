# Критический разбор сайта — Василий Никитин

## Фундаментальные вопросы к идее

**Кто конкретно ваш клиент?**
"Руководители, предприниматели, владельцы" — это 3 разные аудитории с разными болями и бюджетами. Стартапер с 3 сотрудниками и CEO компании на 200 человек читают один и тот же текст и оба думают "это не про меня". Нет сегментации — нет попадания.

**Что именно вы продаёте?**
На сайте 4 услуги + автоматизация. Это 5 разных продуктов. Лендинг, который продаёт всё, не продаёт ничего. Какой главный продукт? Что люди покупают чаще всего?

**Чем вы отличаетесь от 500 других маркетологов, которые тоже написали "ИИ-автоматизация"?**
Сейчас нет ни одного конкретного дифференциатора. "Системный подход" и "фокус на ROI" — это говорят все.

## Критические технические дыры

**Лиды некуда падают.**
Кнопка "Записаться на сессию" ведёт на `#cta` — то есть на себя. Нет Calendly, нет формы, нет воронки. Человек нажал — и ничего не произошло.

**Нет аналитики.**
Ни Яндекс Метрики, ни GA4, ни пикселя. Вы не будете знать, сколько людей зашло, где ушли, что кликали. Оптимизировать нечего.

**Мобильная навигация сломана.**
На мобильных `nav-links` скрывается через `display:none` — и ничего не появляется вместо. Нет бургер-меню. Пользователь не может перейти по разделам.

**Нет meta-тегов.**
Нет `<meta description>`, нет OG-тегов для превью при шеринге в мессенджерах. Когда вы пошлёте ссылку потенциальному клиенту в Telegram — он увидит пустой прямоугольник вместо красивого превью.

## Что обычно забывают — и вы тоже

| Что | Почему важно |
|-----|-------------|
| **Политика конфиденциальности** | Если собираете хоть один email — по закону обязана быть. Крупные компании не работают с подрядчиками без неё. |
| **Юридические реквизиты** | Закупщики в компаниях требуют ИНН/ОГРНИП для договора. Без этого — нет B2B-сделки. |
| **Retargeting-пиксель** | 97% посетителей уйдут без заявки. Без пикселя вы их теряете навсегда — нет возможности "догнать" рекламой. |
| **Страница 404** | Если ссылка сломается — человек видит пустоту браузера, а не ваш сайт. |
| **OG-изображение** | Картинка 1200×630 для шеринга в соцсетях и мессенджерах. |
| **Favicon** | Мелочь, но вкладка без иконки выглядит как незаконченный сайт. |

## Главный структурный риск

Сайт построен по логике **"я предлагаю"**, а не **"вы получите"**. Разница критическая:

```
Сейчас:  "Внедряю ИИ-агентов, которые генерируют контент"
Надо:    "Ваша команда перестаёт писать посты вручную —
          агент делает 30 единиц контента в неделю без вашего участия"
```

Предприниматель думает не про ваши инструменты — он думает про своё время и деньги.

## Что нужно решить до запуска

1. **Выбрать одну главную услугу** для лендинга (остальные — апселл)
2. **Подключить реальную запись** — Calendly или хотя бы форму с автоответом
3. **Добавить хоть один кейс** — без него цифры это просто слова
4. **Метрика/GA4 + пиксель** — до первого посетителя
5. **Починить мобильное меню**
6. **Добавить политику конфиденциальности** — одну страницу

## Рекомендуемая структура сайта (доработанная)

```
Hero (острый оффер + одна главная ЦА)
  ↓
Для кого (уточнение: предпринимателям 20-200 чел, B2B/B2C)
  ↓
Боли → сразу решения (парные блоки: боль + как я её решаю)
  ↓
Услуги (с ценовыми ориентирами или форматами)
  ↓
Кейсы (2-3 истории с цифрами)
  ↓
Процесс (с конкретикой по ИИ-инструментам)
  ↓
Обо мне (фото + 150 слов личной истории + медиа/выступления)
  ↓
Отзывы (3-5 с реальными именами)
  ↓
FAQ (5-7 вопросов)
  ↓
CTA с конкретным оффером + форма
```
